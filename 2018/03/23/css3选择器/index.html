CSS3选择器

发表于 2018-03-23 | 分类于 CSS3

CSS3选择器

class有两个缺点:

  1. class本身没有语义,纯粹用来为CSS服务,属于多余属性

  2. class属性没有把样式与元素绑定起来,针对同一个class,文本框、段落、按钮等均可以使用,显得很混乱,修改时也很不方便

CSS3提倡使用选择器来将样式与元素直接绑定起来

属性选择器

CSS3追加了三个属性选择器

结构性伪类选择器

伪类选择器

  • a:link
  • a:hover
  • a:visited
  • a:active

伪元素选择器

  • :first-line
  • :first-letter
  • :before
  • :after

选择器root,not,empty,target

  • :root指定整个网页
  • :not
  • :empty 元素中内容为空白
  • :target 跳转到target元素后起作用

选择器first-child,last-child,nth-child,nth-last-child

  • first-child
  • last-child
  • nth-child
  • nth-last-child

选择器nth-of-type和nth-last-of-type

nth-child会带来问题,问题产生的原因:nth-child选择器在计算子元素时是连同父元素的所有子元素一起计算的

选择器only-child

某一个父元素中只有一个子元素使用的样式

UI元素状态伪类选择器

指定的样式只有当元素处于某种状态下才起作用,默认状态下不起作用

伪类选择器E:hover,E:active,E:focus

伪类选择器E:enabled,E:disabled

分别用来指定元素处于可用和不可用状态时的样式

伪类选择器E:read-only,E:read-write

伪类选择器E:checked,E:default

伪类选择器E::selection

用来指定元素处于选中状态时的样式

伪类选择器E:invalid,E:valid

当元素内容不能通过HTML5通过使用元素的诸如required,pattern等属性所指定的检查或元素内容不符合元素的规定格式(例如通过使用type属性值为email的input元素来限定元素内容必须为有效的email格式)时的样式

伪类选择器E:required,E:optional
