前端动画总结

发表于 2018-06-15 | 分类于 Web开发

通常在前端中，实现动画的方案主要有以下几种:

  • 使用 setTimeout 与 setInterval 做动画
  • CSS3 transition 与 animation
  • svg 动画
  • requestAnimationFrame

1. 使用 setTimeout 与 setInterval 做动画

1.1 通过 javascript 直接操作 Dom 的样式

其主要思想是通过 setInterval 或 setTimeout 方法的回调函数来持续调用改变某个元素的 CSS 样式以达到元素样式变化的效果。

1.2 canvas 动画

通过 getContext()获取元素的绘制对象，通过 clearRect 不断清空画布并绘制新内容实现页面动画效果。

1.2.1 canvas 动画优势

Canvas 主要优势是可以应对页面中多个动画元素渲染较慢的情况，完全通过 javascript 来渲染控制动画的执行。可用于实现较复杂动画。

1.3 存在的问题

  • javascript 直接操作 Dom 的样式来实现动画通常会导致页面频繁性重排重绘，消耗性能，一般应该在桌面端浏览器。在移动端上使用会有明显的卡顿，不适合做复杂动画；
  • setTimeout 与 setInterval 并不能提供制作动画所需的精确计时机制，他们只是让应用程序能在某个大致时间点上运行代码的普通方法而已；
  • 没有对动画循环的机制做优化；
  • 不考虑绘制动画的最佳时机，而只是一味地以某个大致时间间隔来调用动画循环

2. CSS3 transition 与 animation

2.1 transition

transition 是过度动画。但是 transition 并不能实现独立的动画，只能在某个标签元素样式或状态改变时进行平滑的动画效果过渡，而不是马上改变。

注意：在移动端开发中，直接使用 transition 动画会让页面变慢甚至卡顿。所以我们通常添加 transform:translate3D(0,0,0)或 transform:translateZ(0)来开启移动端动画的 GPU 加速，让动画过程更加流畅。

2.2 animation

animation 算是真正意义上的 CSS3 动画。通过对关键帧和循环次数的控制，页面标签元素会根据设定好的样式改变进行平滑过渡。而且关键帧状态的控制是通过百分比来控制的。

2.3 比较

CSS3 最大的优势是摆脱了 js 的控制，并且能利用硬件加速以及实现复杂动画效果。

3. svg 动画

3.1 比较

SVG 的一大优势是含有较为丰富的动画功能，原生绘制各种图形、滤镜和动画，并且能被 js 调用。html 是对 dom 的渲染，那么 svg 就是对图形的渲染。 但是，另一方面元素较多且复杂的动画使用 svg 渲染会比较慢，而且 SVG 格式的动画绘制方式必须让内容嵌入到 HTML 中使用。CSS3 的出现让 svg 的应用变得相对少了。

4. requestAnimationFrame

requestAnimationFrame 是浏览器针对动画专门优化形成的 API，在性能上比 setTimeout 与 setInterval 做动画要好。
